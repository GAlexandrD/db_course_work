***НАЗВА:*** Зареєструвати користувача у системі

***ID:*** EditProfile
    
***НАЗВА:*** Відредагувати профіль користувача у системі
    
***УЧАСНИКИ:*** Користувач, Система

***ПЕРЕДУМОВИ:*** 

- Користувач авторизований у системі

***РЕЗУЛЬТАТ:*** Користувач має відредагований профіль

***ВИКЛЮЧНІ СИТУАЦІЇ:***

- EditProfile_EX_NotAuthorized Користувач не авторизований у системі
- EditProfile_EX_NotWrittenName Користувач не вписав ім'я та намагається зберегти зміни

***ОСНОВНИЙ СЦЕНАРІЙ:***

1. Користувач натискає на кнопку/іконку "Профіль"(EditProfile_EX_NotAuthorized)
2. Система відкриває профіль користувача
3. Користувач натискає на кнопку/іконку "Редагувати профіль"
4. Система відкриває відкриває форму для редагування профілю
5. Користувач редагує профіль: ім'я, аватар, пошта 
6. Користувач натискає кнопкку/іконку "Зберегти зміни"(EditProfile_EX_NotWrittenName)
7. Система зберігає зміни в профілі користувача
8. Користувач отримує інформацію про успішне редагування профілю
9. Користувач бачить оновлені дані профілю

***ID:*** TicketCreate

***НАЗВА:*** Створити завдання

***УЧАСНИКИ:*** Користувач, адміністратор, система

***ПЕРЕДУМОВИ:*** Існує проєкт

***РЕЗУЛЬТАТ:*** Система створює завдання та додає його до відповідного проєкту

***ВИКЛЮЧНІ СИТУАЦІЇ:***

TicketCreate_EX_NoName - Назву завдання не вказано.\
TicketCreate_EX_NoDesc - Опис завдання не вказано.\
TicketCreate_EX_PressCancel - Натиснута кнопка "Відміна".

***ОСНОВНИЙ СЦЕНАРІЙ:***

1. Адміністратор/користувач проєекту натискає кнопку "Створити завдання".
2. Система відкриває модальне вікно з формою для створення завдання. (TicketCreate_EX_PressCancel)
3. Адміністратор/користувач проєкту заповнює форму (назва, опис, дедлайн, відповідальний, пріоритет, статус, додає файли за потреби).
4. Адміністратор/користувач проєкту натискає кнопку "Створити".
5. Система створює завдання та додає його у відповідний розділ проєкту. (TicketCreate_EX_NoName, TicketCreate_EX_NoDesc)
6. Всі користувачі та адміністратори проєкту бачать завдання.
